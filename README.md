# what_seoul


><center style="font-style: italic">서울 어딘가로 떠나고 싶은 오늘,<br>서울시 실시간 <strong style="font-style: normal">날씨</strong>, <strong style="font-style: normal">혼잡도</strong>와 <strong style="font-style: normal">문화행사</strong>를 장소별로 확인하세요!</center> 

[서울시 공공데이터](https://data.seoul.go.kr/dataList/OA-21285/F/1/datasetView.do)를 활용하여, 서울시 주요 120개 장소의 날씨, 인구, 문화행사 정보를 제공합니다. <br>
서울 실시간 도시데이터 조회 서비스 [왓서울(WhatSeoul)](https://github.com/WhatSEOUL/WhatSeoul) 를 일부 개선하여 새롭게 개발된 웹 서비스입니다.
<br><br>

## 🔆 개요
- 개발 기간: 2025.03.03~ (진행중)
- 개발 인원: 1인
  <br><br>

## ☕ 기술 Stack
- 주 언어 : Java 17
- 프레임워크 : Spring Boot 3.4.3
- DB : MySQL, Spring Data JPA, Querydsl
- 빌드 및 테스트 : Gradle, JUnit
- 템플릿 엔진 : Thymeleaf (+ HTML/CSS, Javascript)
- Cloud & CI/CD: AWS RDS/EC2/CodeDeploy, Github Actions, Nginx
- 인증·인가: Spring Security, JWT, Redis
  <br><br>

## 🫚 브랜치 전략
- main : 실제 서비스 운영 환경에 사용하기 위한 브랜치
- dev : 로컬 개발 환경에서 사용하기 위한 브랜치
    - 하위에 feat/#N과 같이 이슈 번호를 바탕으로 하위 브랜치를 생성하여 작업합니다.
- 하위 브랜치 종류
    - feat : 신규 피쳐 도입
    - fix : 오류 수정 및 개선
    - docs : 문서화 반영
    - test : 테스트 작성
      <br><br>

## 👀 주요 기능
### 🏙️ 서울시 장소별 날씨, 인구, 문화행사 정보 조회 기능
> 서울시 주요 장소 120곳에 대한 **날씨**, **인구**, **문화행사** 정보를 조회합니다.

🔹'지금 서울은?' 메뉴 접속 시, 서울시 주요 장소 120 곳을 지도에 모두 표기합니다. <br>
🔹개별 장소를 클릭할 때마다 '날씨', '혼잡도', '문화행사' 아이콘을 출력하여 아래의 도시데이터를 조회할 수 있도록 합니다. <br><br>
🔸**실시간 날씨 현황** : 특정 장소의 현재 인구, 미세먼지 지표, 강수량 메시지를 조회합니다.<br>
🔸**실시간 인구 현황** : 특정 장소의 현재 혼잡도와 향후 12시간 동안의 예측 혼잡도를 조회합니다.<br>
🔸**문화행사 정보** : 특정 장소에서 진행 중인 문화행사의 장소, 일정, 후기를 조회합니다.<br>

🔹노란색 원형 마커를 클릭해, 진행 중인 문화 행사를 도로명주소별로 모아 조회합니다. 종료된 문화 행사도 함께 조회됩니다. <br>
🔸**한 눈에 서울시의 혼잡도를** : 초록색부터 빨간색까지, 혼잡도에 따라 다른 색상으로 지도에 모든 장소를 출력합니다.

#### 📊 주요 장소 정보 업로드 기능
> 관리자가 '왓서울' 도시데이터의 근간이 되는 공공 API의 업데이트에 유연하게 대응하기 위한 기능입니다.

🔹관리자 로그인 후, 'Setttings' 메뉴에 접속하여 '서울시 주요 장소 정보 목록' 버튼을 클릭합니다.<br>
🔹'서울시 주요 장소 정보 목록' 페이지에서 **서울시 주요 장소 정보 업로드**를 통해 서울시 주요 장소 정보를 나타내는 shapefile을 포함한 압축 파일을 업로드 하고, 이를 데이터베이스에 반영합니다.
<br>

<details>
  <summary>더 자세히 알아보기</summary><br>
  🔹서울시 주요 장소를 지도상에 다각형 형태로 표현하고, 장소별 도시데이터를 저장하기 위해서는 서울시 도시데이터 공공 API가 제공하는 서울시 주요 장소 shapefile이 필요합니다. 이 shapefile은 장소별 장소코드, 카테고리, 이름, 다각형의 좌표 정보를 모두 포함하고 있습니다.<br>
  🔹공공 API의 업데이트에 의해, shapefile에도 변경사항이 발생해 몇몇 장소는 더 이상 도시데이터를 제공하지 않기도, 어떤 장소는 그 이름/카테고리/다각형 좌표가 변경되기도 합니다.<br><br>

🔹'주요 장소 정보 업로드 기능'은, 공공 API 사이터에서 다운로드 할 수 있는 갱신된 shapefile을 업로드하여, 이를 geoPandas를 이용해 geoJSON 형태로 반환합니다.<br>
🔹그리고, 변환된 geoJSON 파일의 각 노드의 장소코드, 카테고리, 이름, 다각형 좌표 정보에 접근합니다. <br>
🔹데이터베이스에 이미 같은 장소코드를 가진 장소가 존재한다면, 현재의 노드와 다른 이름 / 카테고리 / 다각형 좌표 정보를 가지고 있는지 확인하여, 정보를 갱신합니다. <br>
🔹데이터베이스에 이미 같은 장소코드를 가진 장소가 존재하며, 현재의 노드와 같은 이름, 카테고리, 다각형 좌표 정보를 가지고 있다면 정보 갱신을 건너뜁니다. <br>
🔹데이터베이스에 존재하지 않는 장소라면, 신규 정보로 저장합니다. <br><br>


🔹shapefile 포함 압축 파일 업로드에 실패한 경우 : 파일 형식 불일치 등의 이유로 업로드에 실패한 경우, 500 에러 메시지를 alert로 반환합니다.<br>
🔹shapefile 포함 압축 파일 업로드에 성공한 경우 : 아래의 정보를 alert로 반환합니다.<br>
🔹1) 파일이 가진 장소의 개수, 2)새로 저장한 장소의 개수, 3)수정한 장소의 개수, 4)중복되어 갱신을 건너뛴 장소의 개수<br><br>

🔸공공 API 업데이트에 의해 서울시 주요 장소 정보가 변경되더라도, 관리자가 전용 기능을 사용해 대응할 수 있도록 하기 위한 기능입니다.

#### ✏️ 회원정보 수정 기능
> 회원정보 필드별 유효성 검증과 일련의 과정을 거쳐 회원정보를 수정합니다.

🔹아래의 조건을 모두 충족하여 회원정보를 수정합니다. <br>
🔹1. 회원 정보 필드별 **유효성 검사**를 통과한다. <br>
🔹2. 로그인한 계정의 **현재 비밀번호**를 정확히 입력한다. <br>
🔹3. 비밀번호/이메일/닉네임 중 **최소 한 항목에 새로운 값**을 입력한다. (현재 비밀번호를 정확히 입력하더라도 다른 필드를 모두 공란으로 유지할 경우 회원정보 수정에 실패합니다.) <br>
🔹4. 비밀번호/이메일/닉네임을 수정할 경우, **기존과 다른 새로운**값으로 입력한다. <br>
🔹5. 이미 사용 중인 이메일/닉네임은 사용할 수 없다. <br>

🔹회원정보 수정에 성공한 경우 : 로그아웃합니다. <br>
🔹회원정보 수정에 실패한 경우 : 1) 현재 비밀번호를 정확히 입력하였는지, 2) 변경하고자 하는 필드의 유효성(글자수, 형식), 3) 변경하고자 하는 필드의 중복 여부와 관련된 오류 메시지를 출력합니다. <br>

## 📊 도시데이터 저장 스케줄러
> '왓서울'은 1) 서울시 공공데이터 API가 제공하는 데이터의 최단 갱신 주기에 맞추어, 2) 유형별 도시데이터의 특성을 고려하여 도시 데이터를 관리하고 있습니다.

<details>
    <summary>더 자세히 알아보기</summary><br>
'왓서울'은 스케줄러를 사용해 서울시 도시데이터 공공 API를 주기적으로 호출하고, 호출 결과 얻은 XML 데이터를 파싱하여 장소별 날씨, 인구, 문화행사 데이터를 저장하고 있습니다. 이 작업은 스케줄링을 통해 이루어지고 있는데, '왓서울'의 유저에게 가장 최신의 데이터를 정확히 전달하되, 주어진 DB 환경에 적합한 용량의 데이터를 저장하는 방식을 고민하고 있습니다. 이 과정에서, 현재는 각 유형별 데이터의 갱신 주기와 데이터 특성을 고려하기 위해 조금씩 다른 저장 방식을 채택하였습니다.

### 날씨 및 인구 데이터 저장 방식
서울시 도시데이터 공공 API가 제공하는 날씨 현황, 인구 현황, 예측 인구 현황 데이터는 가장 짧을 경우 5분의 간격으로 갱신됩니다. 따라서, '왓서울'은 5분 간격으로 공공 API를 호출하여 서울시 주요 장소 120곳에 대한 이 세 종류의 데이터를 저장합니다. 이는 5분 마다 1,680개의 행을 데이터베이스에 추가하게 되는데, 이 데이터들이 계속 저장된다면 수만 건의 데이터가 누적되는 부하가 일어날 수 있습니다. 따라서 날씨 현황, 인구 현황, 예측 인구 현황 데이터의 경우 적은 양의 최신 데이터만을 남기고 오래된 데이터를 제거하고 있습니다.<br><br>
물론, 향후 날씨 및 인구 정보와 관련하여 유저가 직접 어떠한 데이터를 추가할 수 있도록 서비스가 변화하거나, 날씨 및 인구 정보를 다건 누적하여 통계에 활용하게 된다면 이 방식에 변화가 필요할 것으로 예상합니다.

### 문화행사 데이터 저장 방식
서울시 공공데이터 API가 제공하는 문화행사 데이터의 경우, 날씨 및 인구 데이터와 달리 비교적 긴 주기인 1일을 기준으로 신규 데이터가 추가됩니다. 그러나 관찰 결과, 동일한 문화행사 정보가 미세하게 수정되는 경우가 있었습니다. 따라서, '왓서울'은 서울시 주요 장소 120곳의 문화행사 정보를 하루 간격이 아니라 일 4회 달라진 정보는 없는지, 완전히 새로운 정보는 없는지 확인하는 방식을 선택했습니다. 서울시 도시데이터 공공 API를 호출했을 때, DB에 있는 행사 정보와 동일한 행사 이름, 행사 장소를 가진 데이터가 조회된다면, 행사 기간, 행사 좌표, 썸네일 등 달라진 정보가 없는지 확인하여 갱신합니다. DB에 있는 행사 정보와 완전히 다르다면, 새로이 저장합니다.

### 종료된 문화행사 데이터 관리 방식
모든 문화행사에는 행사 기간이 존재하고, 이는 짧게는 하루에서 길게는 몇개월에 이릅니다. '왓서울'은 종료된 행사의 데이터를 바로 삭제하는 대신, 유보 기간을 두고 있습니다. 일정 시간마다, 종료된지 일정 기간이 경과하였고, 후기가 존재하지 않는 문화행사 정보라면 데이터베이스에서 완전히 제거합니다. 그러나 종료된지 일정 기간이 지나지 않았거나, 후기가 존재하는 경우라면 데이터베이스에서 제거하지 않습니다. 이는 첫째로는 종료된 문화행사 정보 또한 유저에게 필요한 정보일 수 있기 때문이며, 둘째로는 행사가 종료되기 전, 유저가 작성하였던 후기 데이터를 보호하기 위함입니다.
